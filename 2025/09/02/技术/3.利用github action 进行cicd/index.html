GitHub Actions实现Hexo博客CI/CD自动化部署

正文内容

先在github上面新建一个私密仓库
在blog根目录下
依次执行
git init # 初始化
git branch -M master # 将主分支命名为 master，请根据你的仓库默认分支调整
git remote add origin https://github.com/<git名字>/<仓库名>.git # 替换为你的源码仓库 URL
git add . # 添加所有文件
git commit -m “Initial commit” # 添加到缓存区，后面的是备注，可以修改
git push -u origin master
以上步骤完成后，我们顺利的把源码上传到了仓库
然后我们开始建立一个Access Token
git主页->设置->开发者设置->个人访问令牌-> Fine-grained personal access tokens
生成一个秘钥
有效期看自己
注意，Permissions 仓库添加权限Contents和操作 要可读可写
生成之后记得把token保存起来，只显示一次
然后再本地新建一个.github文件夹，注意有个小数点
在.github文件夹新建一个wordflows文件夹
在wordflows文件夹新建一个autodeploy.yml文件，复制一下代码，注意有的需要修改

然后在你源码的项目里面添加GH_TOKEN的这个环境
打开源码的git网址 网页里的设置(不是右上角)->安全->代码空间(## Codespaces secrets)新建一个secret名字就叫做GH_TOKEN，vlaue的就是刚才刚才叫你备份的那个。
如果你忘记备份了，那就要重来一遍了
这是，你在电脑修改之后
可以用以下测试
hexo new “测试”
git add . # 添加所有文件
git commit -m “one” # 添加到缓存区，后面的是备注，可以修改
git push -u origin master
看看在源码git上面的操作里面的工作流程，是打钩的就行，就代表了完成了
