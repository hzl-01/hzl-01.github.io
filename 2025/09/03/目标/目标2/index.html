项目学习计划与模块分解

正文内容

序号/模块 主题 核心内容/技术要点 价值与目的 学习建议 计划学习日期 实际完成日期 备注 (优先级/难点)
项目概述 项目简介与资源 项目功能、视频教程地址、GitHub/Gitee仓库链接 对整个项目建立整体认识，获取所有学习资源。 必读。复制链接，妥善保存。 2025-xx-xx P0
基础准备 技术栈与环境 Vue + Go + Gin + MySQL + Redis, 环境配置 了解构建一个现代化Web项目所需的全套技术组合。 确保本地环境（Go, MySQL, Redis）已准备好。 2025-xx-xx P0 环境是基础
哲学思考 技术与知识 辨析“技术”与“知识”的区别（如：游泳 vs 知道地球是圆的） 建立正确的学习观，明白编程重在掌握“解决实际问题的能力”。 快速理解，指导后续学习心态。 (可选)
核心业务 用户模块 雪花算法(Snowflake) 生成分布式唯一ID 解决传统自增ID在分布式系统下的瓶颈，是分布式系统基础。 精读。理解算法思想及其解决什么问题。 2025-xx-xx P1 分布式基础
Validator 在Gin框架中的高级使用技巧 学习如何高效、优雅地对请求参数进行校验，是Web开发的必备技能。 掌握。 validator库是Gin开发必用。 2025-xx-xx P1 开发必备
限制账号单设备登录 实现常见的风控和安全需求，思考状态管理的方案。 了解实现思路，这是一个很好的业务场景题。 2025-xx-xx P2 场景题
认证授权 认证模式 Cookie-Session 和 Token（JWT） 认证模式的原理与对比 理解Web开发中最核心的两种认证模式及其适用场景。 必会。面试高频考点，必须能说清区别和优劣。 2025-xx-xx P0 面试核心
JWT在Gin中的实战集成 掌握如何在Go Web项目中实现无状态的身份认证，是现代API开发的标配。 重点掌握。亲手实现一遍，会用在几乎所有项目中。 2025-xx-xx P0 实战核心
开发效率 工程化工具 编写Makefile 用Makefile统一管理项目构建、测试、部署命令，极大提升开发效率，体现工程化思维。 必用。为你的项目编写一个Makefile，养成好习惯。 2025-xx-xx P1 工程化
使用Air实现热重载 在开发阶段实现代码实时热更新，提升开发体验和调试效率。 强烈建议使用。开发神器，告别手动重启。 2025-xx-xx P1 提效工具
核心业务 帖子与列表 分页查询 实现数据列表的分页返回，最基本的后端API功能之一。 掌握。理解并实现基于页码和页大小的分页。 2025-xx-xx P1 基础功能
JSON序列化 处理数字精度（如int64大数字）、常用序列化技巧 解决前后端交互中可能遇到的真实问题，写出健壮的代码。 2025-xx-xx P2 细节处理
帖子投票（点赞） 设计并实现投票功能，涉及数据库设计和Redis的使用。 重点掌握。典型的高频读写业务，思考如何保证性能和数据一致性。 2025-xx-xx P0 核心业务
排名算法 学习如何根据投票、时间等因素对内容进行动态排序（如类似Reddit的算法）。 进阶理解。体现业务逻辑复杂度的亮点，能讲出算法原理。 2025-xx-xx P1 项目亮点
API与质量 接口文档 使用Swagger自动生成API文档 学会为项目自动生成可视化、可交互的API文档，便于前后端协作和API测试。 掌握。Go项目生成Swagger文档是行业常见做法。 2025-xx-xx P1 协作必备
性能保障 HTTP压测工具（如wrk, ab） 了解如何对Web服务进行压力测试，量化其性能表现。 了解。至少学会使用一种工具，会看基础指标（QPS, 延迟）。 2025-xx-xx P2 性能意识
限流策略：漏桶算法、令牌桶算法 学习保护服务稳定、防止被流量打挂的核心技术——限流。 重点理解。面试高频考点，理解两种算法的区别和适用场景。 2025-xx-xx P1 面试高频
编程范式 设计模式 Option选项模式 学习一种优雅的Go语言设计模式，用于灵活地初始化复杂对象或配置客户端。 掌握。是编写高质量、易于维护的Go代码的常用技巧。 2025-xx-xx P1 代码质量
性能优化 性能调优 Go pprof性能分析 掌握Go语言自带的强大性能分析工具，学会定位CPU、内存、协程泄漏等性能瓶颈。 必会。性能调优的终极武器，体现你的技术深度。 2025-xx-xx P0 调优神器
部署上线 部署实践 使用Docker部署 学会将Go应用及其依赖（如MySQL, Redis）容器化，实现环境标准化和一键部署。 现代部署首选。必须掌握的技能，写在简历上是亮点。 2025-xx-xx P0 部署核心
多种部署方法 了解除Docker外，传统的二进制文件部署等方式。 拓展视野，了解不同部署方案的优劣。 (可选) P2 了解即可
CI/CD流程 了解现代企业代码发布流程和持续集成/持续部署概念。 了解概念。这是团队协作和工程化的核心流程，能让你更好地融入开发团队。 2025-xx-xx P1 工程化思维
